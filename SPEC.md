# B2 Wortschatz: Требования к приложению

## Назначение

Скрипт читает слова из текстового файла и добавляет их в Google Sheets-словарь B2 по выбранному `Kapitel`, не дублируя уже существующие слова и корректно работая с частями (`Teil` / `часть`).

## Входные данные

Источник: `input.txt` или другой файл, переданный через `--file`.

Формат:

```text
=================================
Существительное:
немецкий термин — русское определение
...
=================================
Глаголы:
...
```

Правила обработки:

- Разделитель блока: строка `=================================`
- Категория задается заголовком блока
- Обрабатываются только строки, содержащие `—`
- Пустые строки игнорируются
- Левая часть от `—` записывается как термин
- Правая часть от `—` записывается как определение

## Поддерживаемые категории

Соответствие категорий листам Google Sheets:

- `Существительное` -> `📗🇩🇪 Fokus Deutsch — B2`
- `Глаголы` -> `📗🇩🇪 Fokus Deutsch — B2`
- `Наречия / Прилагательные` -> `📗🇩🇪 Fokus Deutsch — B2`
- `Фраза: 🗣 REDEMITTEL` -> `🗣️ REDEMITTEL Sprechen`
- `✍️ REDEMITTEL Schreiben` -> `✍️ REDEMITTEL Schreiben`

## Структура Google Sheets

Колонки:

- `A` — номер слова в части, от `1` до `20`
- `B` — термин на немецком
- `C` — не изменяется скриптом
- `D` — определение на русском
- `E` — метка модуля вида `Kapitel X - часть Y`
- `F` — не изменяется скриптом
- `G` — приоритет слова

## Структура Kapitel

На листе существует главный заголовок Kapitel, например:

```text
Kapitel 2 :
```

Под ним располагаются части:

```text
Kapitel 2 : часть 1
Kapitel 2 : Teil 1
```

Правила:

- Каждая часть содержит ровно 20 строк-слотов
- Заголовок части объединяется в диапазоне `A:G`
- Внутри одной секции `Kapitel N` слова должны записываться только в ее части
- Если места в существующих частях больше нет, создается новая часть `max + 1`
- Новая часть должна появляться сразу после последней части этого же Kapitel
- Новая часть должна быть вставлена до следующего Kapitel

Пример:

- после заполнения `Kapitel 2 : часть 14`
- следующей создается `Kapitel 2 : часть 15`

## Логика добавления слов

Для запуска `--kapitel N`:

1. Скрипт ищет нужную секцию `Kapitel N`
2. Находит существующие блоки `часть` / `Teil` внутри этой секции
3. Заполняет свободные строки в существующих частях сверху вниз
4. Если свободных строк нет, создает новую часть `max + 1`
5. Нумерация в колонке `A` всегда идет от `1` до `20` в пределах одной части
6. Колонки `C` и `F` не изменяются
7. Колонка `E` заполняется значением вида `Kapitel X - часть Y`

## Логика дубликатов

Если слово уже существует на соответствующем листе в колонке `B`, новая строка не создается.

Вместо этого обновляется приоритет в колонке `G`.

Шкала приоритета:

- `0` — новое слово
- `1` — встретилось повторно
- `2` — встретилось 2 раза
- `3` — высокий приоритет

Правила:

- дубликаты не должны размножаться при повторных запусках
- повторный запуск с теми же словами должен только обновлять приоритет

## Масштабируемость

Скрипт должен корректно работать при любом объеме данных:

- 20 слов
- 200 слов
- 1000 слов и более

При превышении лимита в 20 слов на часть должны автоматически создаваться новые части.

## Техническая база

- Язык: Python
- Авторизация: Google Sheets API
- Учетная запись: Service Account JSON
- Работа с одним Spreadsheet и несколькими листами внутри него
- Операции записи должны выполняться пакетно (`batch update`)

## Интерфейс запуска

Примеры запуска:

```powershell
.\.venv\Scripts\python.exe add_words.py --kapitel 2 --dry-run
.\.venv\Scripts\python.exe add_words.py --kapitel 2
```

Примечание:

- в этом проекте надежнее использовать явный интерпретатор `.\.venv\Scripts\python.exe`
- команды `python`, `pip` и `py` могут указывать на другое окружение, если в PowerShell уже была активирована другая `venv`

Аргументы:

- `--kapitel` — обязательный номер Kapitel
- `--file` — путь к входному файлу
- `--config` — путь к конфигурации
- `--dry-run` — запуск без записи в Google Sheets

## Требования к устойчивости

Скрипт не должен:

- путать `Kapitel 2` и `Kapitel 12`
- записывать слова в конец всего листа вместо нужного Kapitel
- вставлять пустую строку после каждого слова
- ломать структуру существующих частей
- создавать дубликаты слов в колонке `B`

Скрипт должен:

- корректно работать при многократных запусках
- сохранять структуру частей внутри каждого Kapitel
- создавать новую часть только при реальном отсутствии свободных слотов

## Рекомендации по тестированию

- сначала запускать с `--dry-run`
- для проверки логики использовать копию таблицы
- после подтверждения результата запускать боевой режим без `--dry-run`
